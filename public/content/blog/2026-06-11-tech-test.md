This entry is mostly what it says - a test for loading multiple markdown files!  Because this project is my attempt at a minimal blogging site, I'm doing a lot of shonky coding from first principles, to whit;
* The main static site is written in React and Typescript, and served up with Vite.
* Blog entries and any other big content I want to add are in seperate markdown files with some yaml metadata headers.
* At build time I use a script to take these markdown files and generate a metadata file, which the React code can deal with - this mostly tells it what blog entries are present, along with ordering.
* When a blog entry is selected, via route or link, react-markdown-parser is invoked to generate html.
* Whenever I push a commit to GitHub, including the markdown, GitHub Actions are invoked to generate the metadata, build the site and serve it to Pages.

At least that's the plan.  The current status of this is publically available [here](https://github.com/MikaelaIrish/blorg).
